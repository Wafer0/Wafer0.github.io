Artifact 5 — STM32G491 Bare-Metal Drivers (Micro on Tether)

An ENEE440 (Spring 2025) embedded systems project: a “Micro on Tether” firmware stack on the STM32G491 Nucleo, built from scratch in C and assembly with register-level peripheral drivers, a serial command protocol, PC-side test automation, and full user and maintainer documentation.

Highlights

  • No HAL smokescreen: Peripheral bring-up uses dedicated headers and assembly sources per block (GPIO, RCC, USART, timers, SPI, flash, etc.) so every register write is visible in the repo.
  • Device-oriented firmware: MoT_devices/ splits drivers by subsystem (GPIO, DAC, ADC, TIM2/PWM, SPI, external W25QXX flash, EXTI, plus provided LED/button devices) with a small MoT command dispatcher in MoT_main03.c.
  • Repeatable PC tests: testterm.exe drives scripted serial sessions; the repo ships demo scripts and golden *_result.txt captures for acceptance-style regression.
  • Documentation as deliverable: A technical User Manual for operators and a Maintainer Document for architecture, design choices, and flow — written to match how the code is actually organized under ENEE440S25_PROJECT/.
  • Tooling narrative: README covers Eclipse import, debug launch, COM-port test procedure, and per-device demo expectations (scope where needed).

Context & reflection

The course required a working product: serial protocol, several peripherals, and scripts a new user could step through. Writing the manuals next to the firmware kept the command list and directory layout honest; a vague section usually meant the code boundary was vague too. It is a small-memory, assembly-heavy stack compared with my RTL projects, and it was good practice for documentation meant for someone else.
