Artifact 1 — Advanced Standard-Cell Placer

A multi-threaded VLSI floorplanner written in Rust that uses a B*-tree representation and simulated annealing to place standard cells, with optional timing- and congestion-aware cost terms.

Architecture

Highlights

  • B*-tree floorplan representation with O(n2) packing via a contour structure, following Chang et al. (DAC 2000).
  • Multi-objective cost function: area + half-perimeter wirelength, with optional timing (Elmore-delay model) and congestion (grid-based demand) terms.
  • Multi-threaded simulated annealing using Rayon — eight independent runs with best-of-N selection, yielding 5–8× speedup and 8–12% better solutions than a single run.
  • Validated on the MCNC B10–B300 benchmarks with 89–95% area utilization and zero geometric overlaps.
  • Modular binaries: a placer, a standalone validator, an SVG visualizer, and a benchmark harness with JSON summaries for downstream EDA integration.

Context & reflection

I built this placer as an EDA-tooling deep-dive: rather than treat place-and-route as a black box, I wanted to write the optimizer myself and understand why a B*-tree is the right encoding for non-slicing floorplans. The codebase implements the classic Chang/Wong B*-tree representation with simulated annealing, then extends the cost function with a timing-aware term (Elmore-delay propagation along critical paths) and a congestion-aware term (grid-based demand tracking) and parallelizes the search with Rayon to overcome local optima. It targets the MCNC benchmarks B10 through B300 and produces validated, SVG-visualized floorplans alongside JSON summaries that can be consumed by other tools.

Implementation-wise, the hard parts were turning the B*-tree papers into contour packing and a stable simulated-annealing loop, then bolting on timing and congestion terms without breaking the geometry checks. I split Rust binaries for place, validate, render, and bench so the flow stays testable. The placer is the project I point to when someone asks how I learn a new domain: read the source papers, code the minimal correct version, validate on MCNC, then optimize.
